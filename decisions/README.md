# Decision Verticals

Use these files to move from ideas to a construction-ready bathroom plan.

For this project, **MVP means the smallest complete plan that can be built safely without causing expensive rework**. It does not mean reducing waterproofing quality, skipping inspections, or selecting the cheapest materials.

Use [DUMP.md](../DUMP.md) to process unstructured project thoughts, research, pricing, and decisions into these verticals.

## Decisions To Make First

Complete verticals 1 through 3 before making major purchases or starting demolition.

The minimum pre-demo decision set is:

- Confirm what is in scope and what can be deferred
- Set the working budget and contingency
- Decide which work requires licensed or specialized trades
- Capture all room, shower, vanity, drain, and fixture measurements
- Confirm the intended shower footprint, entry type, bench removal, storage, and drain plan
- Select the complete waterproofing system and compatible drain strategy
- Define a post-demolition inspection checkpoint before rebuilding begins

## MVP Guardrails

Do not defer:

- Waterproofing system selection and installation requirements
- Shower-floor slope, drainage, and flood testing
- Repair of water damage, mold, rot, framing, or subfloor defects
- Required plumbing and electrical code compliance
- Effective bathroom ventilation
- Compatibility between hidden systems and finish materials

Can be simplified or deferred:

- Decorative shower niche or specialty storage
- Premium tile patterns and accent tile
- High-end mirrors, lighting, and hardware
- Cabinet repainting if the existing finish can remain temporarily
- Nonessential layout changes

## How To Use These Files

1. Work through the verticals in order.
2. Record decisions and links directly in each file.
3. Mark the vertical complete only when its MVP exit criteria are satisfied.
4. Revisit downstream verticals when an earlier decision changes.
